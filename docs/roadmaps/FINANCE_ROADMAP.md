# Crazy Factory 金融核心玩法路线图（Finance-First Core Loop）

> 目标：将金融从“中后期分支”提升为**全局第一核心循环**，让玩家每 30~90 秒都在做金融相关决策，并驱动产线、订单与 Prestige 节奏。

## 1. 核心定位（重定义）
- **玩法主轴**：金融是主循环中枢，不再是附属收益模块。
- **决策优先级**：优先处理“资金配置 → 风险控制 → 对冲策略”，再决定产线与订单投入。
- **体验目标**：玩家能明确感知“金融决策 = 本局效率与节奏差异的主要来源”。

---

## 2. 北极星指标（Finance-First KPI）
- **决策密度**：每 3 分钟至少出现 2 次有效金融决策（投入/提取/升级/对冲/切换策略）。
- **策略分化**：同样 15 分钟时长下，金融策略路径之间应形成可解释差异（收益、波动、风险暴露）。
- **可读性**：玩家能从 UI 直接看到“当前收益由哪 3~4 层构成”，且知道下一步最优动作候选。
- **不失控约束**：金融强度虽为核心，但不得破坏无金融玩法的基本通关路径。

---

## Phase B（近期）：金融驱动全局节奏
**目标**：由“金融收益”升级为“金融驱动玩法节拍”。

### 功能包
- [x] 市场状态机（平稳 / 过热 / 回调 / 恐慌）接入主 HUD。
- [x] 新增金融策略切换（稳健 / 平衡 / 激进）并影响订单与产线收益权重。
- [x] 引入对冲动作（短时降低收益换取波动收敛）。
- [x] 订单加入金融标签（融资单 / 套保单 / 结算单），优先由金融状态触发。

### 约束
- [x] 任一市场状态都必须提供“至少一个低风险动作 + 一个高风险动作”。
- [x] 激进策略不得长期稳定优于平衡策略（必须有回撤窗口）。

### 验证
- [x] 增加 `scripts/finance_checks.js`：回撤上限、峰谷切换、策略收益分位。
- [x] `balance_runs_20m` 增加 3 类金融画像（稳健/平衡/激进）并输出对比。

## Phase C（中期）：金融成为 Prestige 前后核心资源系统
**目标**：让 Prestige 变成“金融阶段结算与再配置”。

### 联动设计
- [x] Prestige 前：金融提供冲刺窗口（短时高收益+高波动）。
- [x] Prestige 后：保留部分金融元进度（风控天赋、信用等级或市场情报）。
- [x] 产业链健康度反向影响融资成本/信用等级，形成双向约束。

### DoD
- [x] 玩家在一次 30 分钟流程内，至少经历 2 次“金融策略切换”并影响 Prestige 时机。
- [x] 三种路线（产线优先 / 金融优先 / 联动均衡）都可达成首轮 Prestige。

---

## 4. 参数治理（金融核心参数区）
- `FINANCE_BASE_APR`：基础收益地板。
- `FINANCE_TIER_APR_BONUS`：风控成长斜率。
- `FINANCE_CYCLE_AMPLITUDE`：波动振幅。
- `FINANCE_MAX_SHARE_OF_MAINLINE`：金融相对主线上限。
- `FINANCE_MIN_CAP_GPS`：低主线阶段保底可感知收益。
- 投入/提取比例与冷却：决策频率与流动性。

> 规则：每次迭代最多改 2 个核心参数，并附 15/20/30 分钟脚本输出。

## 6. 风险与止损（Finance-First 下的护栏）
- **风险 A：金融过强导致其他系统沦为装饰**
  - 止损：下调 `FINANCE_MAX_SHARE_OF_MAINLINE`，提高高风险策略回撤。
- **风险 B：金融信息过载导致理解成本高**
  - 止损：保留 3 层核心展示 + 高级详情折叠。
- **风险 C：高频决策造成操作疲劳（已落地半自动策略）**
  - 止损：提供“半自动策略执行”与低频提醒模式。
