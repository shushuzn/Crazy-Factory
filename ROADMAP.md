# 自动化工厂（Idle/Incremental）路线图（精简版）

## 目标
将原型推进为可持续迭代的可玩版本：有中期目标、可验证数值、可维护代码结构。

## 里程碑

### M1：核心循环稳定化
- [ ] 数值层重构（生产/购买/渲染解耦）
- [x] 固定步长 Tick（减少帧率波动影响）
- [x] 渲染层缓存 DOM 引用
- [x] 数值显示规范化（K/M/B）
- [ ] 前 10 分钟成长曲线调优

### M2：进度与可玩性增强
- [x] 批量购买（x1/x10/x100/Max）
- [x] 升级系统（全局/建筑增益）
- [x] 本地存档（自动保存 + 手动重置）
- [x] 离线收益（封顶结算）
- [x] 成就与新手目标提示

### M3：中期循环扩展
- [ ] 新建筑层级
- [x] 解锁条件系统（建筑/升级前置）
- [x] 软重置（Prestige / RP）
- [ ] 经济曲线二次调优
- [ ] 音效与关键反馈动效

### M4：工程化与发布准备
- [ ] 模块化拆分（ui/economy/save）
- [ ] 单元测试（价格、批量购买、离线、Prestige）
- [ ] 埋点与调试面板
- [ ] 长时间运行性能巡检
- [ ] 发布页与版本日志

## 当前版本已落地能力
- 固定步长模拟 + 自动保存
- 建筑购买与批量购买
- 升级系统、成就系统、新手目标
- 解锁条件系统（累计齿轮 / 前置升级 / RP 门槛）
- 成就奖励联动（齿轮/RP）
- 离线收益结算
- Prestige（RP 永久总产出加成）

## 下一步优先级
1. 调整 Prestige 与升级数值曲线
2. 增加建筑专属升级与解锁条件
3. 为核心经济逻辑补齐最小自动化测试
4. 拆分单文件，降低后续维护成本

## 更新规则
- 每次迭代只更新三块：`已完成`、`待处理`、`下一步`。
- 文档保持短句、可勾选、可追踪。
