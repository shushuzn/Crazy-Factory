# 自动化工厂（Idle/Incremental）迭代路线图

## 目标
将当前原型从“可运行 Demo”升级为“可持续扩展的游戏核心”，并逐步完善内容、体验与留存。

## 里程碑总览

### M1：核心循环稳定化（1-2 天）
- [ ] 数值层重构为 `GameState + Systems`（生产、购买、渲染解耦）
- [x] 增加统一 Tick（固定步长），避免高帧率下显示抖动
- [x] UI 渲染优化（缓存 DOM 引用，减少频繁 query）
- [x] 金额显示规范化（K/M/B 后缀 + 精度规则）
- [ ] 基础平衡校准（首 10 分钟成长曲线）

**验收标准**
- 10 分钟内可稳定体验“点击 → 首个自动化 → 复购加速”的正反馈
- FPS 与数值增长在不同设备上表现一致

---

### M2：进度与可玩性增强（2-3 天）
- [x] 新增“批量购买”（x1/x10/x100/Max）
- [x] 引入“升级系统”（全局倍率、建筑专属倍率）
- [x] 本地存档（自动保存 + 手动重置）
- [x] 离线收益（按离线时长上限结算）
- [ ] 成就与提示（新手引导 + 阶段目标）

**验收标准**
- 玩家刷新页面后可恢复进度
- 离线回归有明显补偿且不会破坏平衡

---

### M3：中期目标与内容扩展（3-5 天）
- [ ] 增加新建筑层级（例如：锻造线、机械臂、工厂 AI）
- [ ] 解锁条件系统（达到总产出/总齿轮后开放）
- [ ] 软重置（Prestige）：重置换取“研究点”永久加成
- [ ] 经济曲线二次调优（避免中期停滞）
- [ ] 基础音效与反馈动效（购买成功、解锁、里程碑）

**验收标准**
- 玩家在 30-60 分钟内拥有清晰中期目标
- Prestige 前后体验有“重开更快”的明确感知

---

### M4：工程化与发布准备（2-3 天）
- [ ] 模块化拆分（`ui.js` / `economy.js` / `save.js`）
- [ ] 增加单元测试（价格公式、批量购买、离线结算）
- [ ] 埋点与调试面板（产出、购买率、卡点）
- [ ] 性能巡检（长时间挂机内存稳定性）
- [ ] 发布页与版本日志（Changelog）

**验收标准**
- 核心经济逻辑具备自动化测试覆盖
- 连续运行 2 小时无明显性能退化

## 数值设计建议（初版）
- 建筑价格：`price = floor(basePrice * 1.15 ^ owned)`（保留）
- 建筑效率：中后期建筑需保持 5-8 倍性价比阶梯，防止低级建筑完全失效
- 升级节奏：每 2-3 分钟出现一次“可达成目标”，避免空窗
- 离线上限：建议 8-12 小时封顶，防止离线碾压在线体验

## 风险与对策
- **风险：** 前期点击过慢导致流失  
  **对策：** 提高前 1-2 分钟点击收益或赠送首个建筑折扣
- **风险：** 中期卡关  
  **对策：** 提供阶段性升级与任务奖励兜底
- **风险：** 存档异常  
  **对策：** 增加存档版本号与回退机制
