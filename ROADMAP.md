## 📅 阶段一：优化与完善（当前阶段）

**时间范围**: 2026-06 → 2026-07  
**目标**: 提升稳定性，完善用户体验

### 1.1 数值平衡优化

#### 🔴 P0 - 高优先级

**任务 1.1.1: 提升市场稳定性**
- **问题**: 市场稳定性仅 1.2%（目标 5-8%）
- **方案**: 
  - 调整 `market-system.js` 中的波动参数
  - 优化宏观事件触发频率和后效
  - 校准利率前瞻命中率
- **验收标准**: 市场稳定性 > 5%
- **负责人**: 数值策划 + 工程师
- **预计工期**: 3-5 天

**任务 1.1.2: 减少瓶颈期**
- **问题**: 瓶颈期 1070 秒（目标 < 600 秒）
- **方案**:
  - 运行 `scripts/run_momentum_balance_check.js`
  - 识别并缩短瓶颈期
  - 调整价格曲线或收益公式
- **验收标准**: 瓶颈期 < 600 秒
- **负责人**: 数值策划 + 工程师
- **预计工期**: 5-7 天

**任务 1.1.3: 评估离线收益**
- **问题**: 离线收益 57%（需评估是否合理）
- **方案**:
  - A/B 测试不同离线收益比例
  - 收集玩家反馈
  - 决定是否调整
- **验收标准**: 做出明确决策（调整或保持）
- **负责人**: 产品经理 + 数据分析
- **预计工期**: 3-5 天

#### 🟡 P1 - 中优先级

**任务 1.1.4: 微调增长曲线**
- **方案**: 基于 `output/tuning_report.json` 的 top-k 参数进行微调
- **验收标准**: 风险调整收益保持在 80-90% 区间
- **预计工期**: 5-7 天

### 1.2 用户体验改进

#### 🔴 P0 - 高优先级

- **负责人**: UX 设计师 + 前端工程师

**任务 1.2.2: 创建 ROADMAP.md**
- **问题**: README.md 引用了 ROADMAP.md 但它不存在
- **方案**: 创建本文件 ✅ **已完成**
- **验收标准**: ROADMAP.md 完整且准确
- **预计工期**: 1 天 ✅

#### 🟡 P1 - 中优先级

**任务 1.2.3: 视觉反馈优化**
- **方案**:
  - 购买、升级、成就解锁动画

  - 收益数字动画（跳动、颜色变化）
- **验收标准**: 玩家满意度调查 > 4.0/5.0

**任务 1.2.4: UI 信息架构优化**
- **问题**: 30+ 系统模块，信息密度高
- **方案**:
  - 重新组织面板布局
  - 添加可折叠/展开的模块
  - 优化移动端显示
- **验收标准**: 新玩家能在 30 秒内找到核心功能
- **预计工期**: 7-10 天

---

### 1.3 性能优化

#### 🟡 P1 - 中优先级

**任务 1.3.1: 渲染性能优化**
- **方案**:
  - 使用 `requestAnimationFrame` 优化动画
  - 虚拟滚动（建筑列表增长时）
  - 批量 DOM 更新
- **验收标准**: FPS 稳定在 60（之前可能有掉帧）
- **负责人**: 前端工程师
- **预计工期**: 5-7 天

**任务 1.3.2: 内存优化**
- **方案**:
  - 事件日志轮转（防止无限增长）
  - 存档压缩优化
  - 图片资源懒加载
- **验收标准**: 长时游戏（> 2小时）内存增长 < 100MB
- **预计工期**: 5-7 天

#### 🟢 P2 - 低优先级

**任务 1.3.3: 增强调试面板**
- **方案**:
  - 在 `?debug=1` 面板添加更多指标
  - 自动上传性能报告（可选）
  - CI 集成性能门禁
- **验收标准**: 开发者能快速定位性能问题
- **预计工期**: 3-5 天

## 🚀 技术债务与风险

### 技术债务

1. **JavaScript 模块化**
   - 当前使用 `<script defer>` 加载
   - **建议**: 迁移到 ES Modules + Webpack/Vite 打包
   - **优先级**: P2（低）
   - **预计工期**: 14-21 天

2. **代码组织**
   - 部分系统模块过大（`game.js` 70KB）
   - **建议**: 拆分和重构
   - **优先级**: P2（低）
   - **预计工期**: 21-28 天

### 风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| 系统复杂度过高 | 新玩家流失 | 中 | 优化新手引导、简化 UI |
| 性能瓶颈 | 玩家流失 | 低 | 定期性能监控和优化 |
| 过度商业化 | 玩家反感 | 中 | 避免 Pay-to-Win、注重公平性 |
| 移动端适配失败 | 无法扩展用户群 | 低 | 充分测试、渐进式适配 |

## 📝 贡献指南

### 如何贡献

1. **Fork 仓库**
2. **创建功能分支** (`git checkout -b feature/AmazingFeature`)
3. **提交更改** (`git commit -m 'Add some AmazingFeature'`)
4. **推送到分支** (`git push origin feature/AmazingFeature`)
5. **开启 Pull Request**

### 代码规范

- **JavaScript**: 遵循 Google JavaScript Style Guide
- **CSS**: 使用 Tailwind CSS 优先
- **提交信息**: 使用 Conventional Commits

### 测试要求

- 所有新功能必须包含单元测试
- 运行 `npm test` 确保所有测试通过
- 目标测试覆盖率 > 80%
