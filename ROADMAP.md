# Crazy Factory 发展路线图（2026 项目化版）

> 愿景：把当前原型打造成“可长期迭代、可持续平衡、具备明显操作爽感”的轻量放置游戏项目。

## 0. 北极星目标（North Star）
- **产品目标**：形成「点击爽感 → 自动化扩张 → 中期策略 → Prestige 回圈」的稳定闭环。
- **体验目标**：新玩家 3 分钟上手，10 分钟进入策略抉择，30 分钟仍有明确追求。
- **工程目标**：从单文件原型演进为可维护模块化结构，支持快速试错与回归验证。

---

## 1. 年度阶段路线（按季度推进）

## Q1：原型稳定化（Foundation）
**阶段目标**：稳定核心经济循环，建立最小测试与调参能力。

### 核心交付
- [ ] 抽离经济纯函数（价格、收益、离线、Prestige）。
- [x] 增加 `saveVersion` 与存档迁移逻辑。
- [x] 建立最小自动化校验脚本（公式边界 + 回归样例）。

### 退出标准（DoD）

- [ ] 关键公式可一键回归。
- [ ] 老存档可识别版本并安全处理。

---

## Q2：玩法深度化（Progression）
**阶段目标**：让“策略选择”成为中期乐趣来源，而不是单一堆倍率。

### 核心交付
- [x] 生产订单 MVP（已落地）
- [x] 订单池分层（新手/中期/高风险高收益）
- [ ] 建筑专属升级树（每建筑至少 2 个分支）
- [ ] Prestige 2.0（新增非线性永久加成）

### 退出标准（DoD）
- [ ] 10~20 分钟至少出现 3 次有效决策点。
- [ ] 不同成长策略收益差距可控（避免单一最优解）。
- [ ] 订单刷新消耗在经济上“可感知但不惩罚”。

---

## Q3：Game Feel 与内容扩展（Juice）
**阶段目标**：强化反馈层，提升“动作价值感”和“奖励峰值”。

### 核心交付
- [x] 基础反馈（浮字/弹跳/脉冲，已落地）
- [x] 连击反馈系统（分段强度、节奏音效）
- [ ] 高价值事件演出（任务完成/订单完成/Prestige）
- [x] 音频开关与混音分层（UI / Reward / System）

### 退出标准（DoD）
- [ ] 所有关键操作都有明确视听反馈。
- [ ] 反馈参数一处可调，支持快速 A/B。
- [ ] 反馈并发受控，不出现刷屏或明显掉帧。

> 性能提示：当前浮字基于 DOM 动态创建；若后续触发频率继续提升，建议切换对象池或 Canvas 粒子层，避免频繁内存分配。

## Q4：工程化与发布（Ship）
**阶段目标**：完成从“可玩原型”到“可发布版本”的交付链路。

### 核心交付

- [ ] 调试面板（资源注入、倍率覆写、存档快照）

### 退出标准（DoD）
- [ ] 单功能改动不影响整局核心循环。
- [ ] 发布前检查清单可执行、可追踪。
- [ ] 至少完成 1 次完整小版本发布演练。

## 2. 指标看板（每次迭代都要看）

### 体验指标
- [ ] 首个任务 3 分钟完成率 ≥ 80%
- [ ] 10 分钟触发一次高价值奖励率 ≥ 70%
- [ ] 首次 Prestige 达成时间在目标区间（建议 18~30 分钟）

### 经济指标
- [ ] 15 分钟内无超过 90 秒“无操作可做”时间窗
- [ ] Prestige 后回到重置前产能耗时 ≤ 40%
- [ ] 订单刷新成本在 3 次连续刷新后仍可恢复

### 技术指标
- [ ] 关键公式回归通过率 100%
- [ ] 30 分钟运行无明显内存膨胀
- [ ] 存档导入失败可恢复，不导致坏档

## 5. 主要风险与处理策略
- **风险 A：数值膨胀导致玩法失真**
  - 策略：乘区拆层、软上限、一次迭代仅改 1~2 个关键参数。
- **风险 B：反馈太多导致性能和视觉疲劳**
  - 策略：反馈分级（普通/关键）、并发上限、低配降级开关。
- **风险 C：单文件持续膨胀降低迭代速度**
  - 策略：先抽纯函数，再拆模块；每一步都保持可运行版本。
