# Crazy Factory 金融发展路线图（Finance Progression Track）

> 目标：把当前“资金池 + 风控等级”从 MVP 演进为可长期调优、可读性强、可和产业链/订单协同的中后期玩法主线。

## 1. 设计北极星
- **玩法定位**：金融不是“被动送钱器”，而是“资源时机管理器”。
- **策略目标**：让玩家在“扩产线 / 做订单 / 投金融”之间持续做权衡。
- **体感目标**：每次金融操作都应有明确反馈（收益变化、风险变化、阶段里程碑）。

---

## 2. 三阶段发展图

## Phase A（当前~短期）：可玩 MVP 稳定化
**目标**：把基础金融循环跑顺，确保不会破坏主经济节奏。  
**已具备**：
- 资金池投入/提取。
- APR 随风控等级变化。
- 风控升级 RP 消耗。

**下一步交付（建议 1~2 周）**
- [x] 金融收益拆层展示（基础 APR / 周期波动 / 风控加成）。
- [x] 投入与提取增加最小冷却（防止高频抖动操作成为最优）。
- [x] 增加金融日志标签（投入、提取、升级）便于复盘。

**DoD**
- [ ] 15 分钟内，金融分支能提供“可感知但不碾压”收益。
- [ ] 未参与金融玩法时，主线仍能稳定推进。

---

## Phase B（中期）：风险与事件驱动
**目标**：从“固定收益”升级为“风险收益曲线 + 事件决策”。

### 功能包
- [ ] 市场状态机（平稳 / 过热 / 回调 / 恐慌）。
- [ ] 金融事件卡（短期 buff/debuff，提供对冲选项）。
- [ ] 风控分支（稳健/激进）与差异化收益曲线。

### 平衡约束
- [ ] 风险越高，收益波动越大；最大收益不能长期压制产业链路线。
- [ ] 任一市场状态都应存在可操作选项，避免“纯看运气”。

### 验证
- [ ] 新增 `finance` 维度断言：回撤上限、极端波动边界、收益上限。
- [ ] 在 `balance_runs_20m` 中加入“金融偏好画像”对比输出。

---

## Phase C（中后期）：与订单/产业链深度联动
**目标**：把金融系统并入全局循环，形成“订单-产业链-金融”三角驱动。

### 联动设计
- [ ] 订单出现“融资单 / 对冲单 / 结算单”模板。
- [ ] 产业链健康度影响金融信用等级（APR 或手续费）。
- [ ] Prestige 后保留部分金融元进度（如风控天赋点）。

### 体验设计
- [ ] 高价值金融事件触发专属高光模板（与普通奖励分层）。
- [ ] 关键决策提供“预估收益区间”文案，降低黑箱感。

### DoD
- [ ] 30 分钟内至少出现 2 次“是否切金融策略”的有效决策点。
- [ ] 3 条成长路线（产线优先 / 金融优先 / 联动均衡）都能通关首轮 Prestige。

---

## 3. 参数治理清单（可直接调）
- `FINANCE_BASE_APR`：金融底收益。
- `financeTier` 每级增益：风控成长斜率。
- 投入比例 / 提取比例：资金流动速度。
- 周期波动幅度：高风险体验强度。
- （后续）市场状态权重与持续时长。

> 规则：每轮最多调整 2 个金融核心参数，并附脚本输出截图/日志。

## 5. 风险预警与止损规则
- **风险 A：金融收益压制主线产线**
  - 止损：降低 APR 增长斜率，增加提取成本或冷却。
- **风险 B：波动过强导致挫败**
  - 止损：设置回撤上限与保底收益。
- **风险 C：文案黑箱导致玩家不理解**
  - 止损：拆分收益来源展示 + 关键操作前后对比提示。

---

## 6. 执行建议（下一迭代）

- Sprint 2：市场状态机 MVP + 2 张事件卡。
- Sprint 3：订单金融联动模板 + Prestige 元进度试点。
