# AGENTS.md (practical-push)

目标：每轮交付 1 个最小可验证增量(MVI)，且必须产生“可见能力推进”；同时维护 ROADMAP.md（更新 + 续写 + 清理）。

R1 任务选择：
- 若有 (NEXT) 必做。
- 否则从 TODO 选“依赖最少 + 最易验证 + 可见能力推进”的任务设为 NEXT。
- 每轮仅 1 个任务。

R1a 推进强制：
本轮必须新增“可见能力”，否则不得 DONE。
可见能力包括：
- 新接口 / 新命令
- 新用户流程
- 新测试覆盖真实行为
- 完成一个端到端最小功能切片(E2E)

R1b SmallFix 限制：
以下不算推进任务（禁止单独作为一轮）：
- 重构
- 重命名/格式化
- 补日志
- 文档润色
除非其为推进任务的必要阻塞步骤，并在 Verify 中说明。

R1c 连续熔断机制：
若连续 2 轮未产生可见能力推进，
第 3 轮必须完成一个端到端最小功能切片。

R2 验证：
无证据不得 DONE。
优先：测试 > 运行结果 > 接口/日志示例。
证据必须证明新增能力可用。

R3 更新路线图：
- 任务 -> DONE，写日期 + 验证证据。
- 新问题 -> (NEW)，写原因 + 验收标准。
- 保证 NEXT 仅 1 个。

R4 续写：
- 每个 Milestone 保持 3~8 个任务。
- 模糊或 >1天任务必须拆分。
- 不得凭空发明不存在的模块。

R5 清理：
- 删除重复/过时任务。
- 无验收标准任务补齐或删除。
- TODO ≤ 20。
- DONE 仅保留最近 15 条，其余汇总。

输出限制：
- 总输出 ≤ 200 行。
- 每段 ≤ 7 条。
- 不讲原理，不闲聊。
- 禁止输出完整 ROADMAP.md。
- 仅输出 RoadmapPatch(diff)。

输出格式固定：

[Task]
id / acceptance

[Do]
- bullets

[Verify]
cmd + result

[RoadmapPatch]
(diff only)

[Next]
next_id
